[~] FOLD-048 — Context menu cartelle: Share Folder e Open All Connections
------------------------------------------------------------------------------
  Priorita: MEDIA
  Dipendenze: SHR-005 (completato), FOLD-001 (completato)

  DESCRIZIONE:
  Aggiungere due nuove azioni al menu contestuale delle cartelle:
  1. "Open All" — apre un tab per ogni connessione nella cartella. Se la
     cartella contiene sottocartelle, mostra un dialog che chiede all'utente
     se includere anche le connessioni delle sottocartelle (con il conteggio
     per ciascuna opzione). Se il totale supera 5 connessioni, mostra un
     dialog di conferma con il conteggio prima di procedere.
  2. "Share Folder" — condivide tutte le connessioni contenute nella cartella
     e ricorsivamente nelle sottocartelle con un altro utente. Poiche' il
     modello SharedConnection e' per-connessione, questa operazione crea una
     condivisione individuale per ciascuna connessione trovata. Richiede un
     nuovo endpoint backend per batch share (condivisione multipla atomica).

  DETTAGLI TECNICI:
  Frontend:
    - treeHelpers.tsx (FolderItem): aggiungere due nuove voci al context
      menu della cartella, dopo "New Subfolder" e prima di "Rename":
      * "Open All" con icona OpenInNew o PlayArrow — visibile sempre
      * "Share Folder" con icona FolderShared — visibile solo se la cartella
        contiene connessioni di cui l'utente e' owner
    - Estendere FolderItemProps con due nuovi callback:
      * onBulkOpen: (folderId: string) => void
      * onShareFolder: (folderId: string, folderName: string) => void
    - ConnectionTree.tsx: implementare i handler per i due nuovi callback:
      * handleBulkOpen(folderId):
        1. Raccogliere le connessioni dirette della cartella
        2. Controllare se ci sono sottocartelle
        3. Se ci sono sottocartelle: mostrare dialog "Include subfolder
           connections?" con opzioni "This folder only (N connections)" e
           "Include subfolders (M connections)" + Cancel
        4. Se il totale scelto > 5: mostrare conferma "Open N connections?
           This will create N new tabs."
        5. Per ciascuna connessione, chiamare openTab(conn)
      * handleShareFolder(folderId, folderName): impostare lo stato per
        aprire il ShareFolderDialog
    - Aggiungere funzione helper in treeHelpers.tsx o utils:
      collectFolderConnections(folderId, connections, folders, recursive):
        ConnectionData[] — raccoglie le connessioni in una cartella,
        opzionalmente ricorsivo nelle sottocartelle
    - Creare client/src/components/Dialogs/ShareFolderDialog.tsx:
      * Simile a ShareDialog.tsx ma per condivisione batch
      * Props: open, onClose, folderId, folderName
      * Al mount: calcolare la lista connessioni ricorsive nella cartella
        (attraversare il folder tree dal connectionsStore per trovare tutte
        le connessioni in folderId e nelle sottocartelle)
      * Mostrare un riepilogo: "Share N connections from folder X"
      * Includere lista scrollabile delle connessioni che verranno condivise
        (nome + tipo icona)
      * UserPicker/email per selezionare il destinatario (stesso pattern
        di ShareDialog)
      * Selettore permesso (READ_ONLY / FULL_ACCESS)
      * Al submit: chiamare il nuovo endpoint batch share con tutti i
        connectionId + target + permission
      * Mostrare progresso/risultato: "N of M connections shared
        successfully" (alcune potrebbero fallire se gia' condivise)
    - MainLayout.tsx: montare ShareFolderDialog, gestire stato
      shareFolderTarget: { folderId: string, folderName: string } | null
  Backend:
    - sharing.service.ts: nuova funzione batchShareConnections() che:
      1. Riceve: actingUserId, connectionIds[], targetUserId o email,
         permission, tenantId
      2. Per ogni connectionId, chiama la logica di shareConnection()
         esistente (validazione permessi, decrypt/re-encrypt credenziali)
      3. Usa Promise.allSettled() per gestire errori parziali (una
         connessione potrebbe fallire senza bloccare le altre)
      4. Restituisce un riepilogo: { shared: number, failed: number,
         alreadyShared: number, errors: Array<{ connectionId, reason }> }
      5. Invia una singola notifica riassuntiva al destinatario
         (es. "User X shared N connections from folder Y with you")
    - sharing.controller.ts: nuovo handler batchShare() con schema Zod:
      { connectionIds: z.array(z.string().uuid()).min(1).max(50),
        target: z.union([z.object({ email: z.string().email() }),
                         z.object({ userId: z.string().uuid() })]),
        permission: z.enum(['READ_ONLY', 'FULL_ACCESS']) }
      Limite max 50 connessioni per batch per evitare abuse
    - sharing.routes.ts: aggiungere POST /api/sharing/batch
      (protetto da authenticate middleware)
    - Aggiungere BATCH_SHARE all'enum AuditAction in schema.prisma
    - Eseguire migrazione Prisma
